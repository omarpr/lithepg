# v1.0 — Public Launch — Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Spec:** `docs/superpowers/specs/2026-05-28-v1.0-public-launch-design.md`

**Goal:** Produce a public-launch-ready macOS distribution path for LithePG: signed/notarized app packaging, release docs, governance templates, light/dark appearance support, and final dogfood/release receipts.

**Architecture:** Keep launch work incremental and credential-safe. First harden local packaging and docs that can be completed without secrets; then add credential-gated signing/notarization wrappers; then add app appearance preference; finally create release receipts and tag only after local gates pass and Omar supplies any required external credentials.

**Tech Stack:** Swift 6.2, SwiftUI, Observation, Swift Testing, shell scripts, GitHub Releases/Homebrew documentation. No telemetry, cloud inference, or bundled model downloads.

**Workflow:** Direct commits to `main`; no PRs unless Omar asks. Code tasks use strict TDD. Docs/script-only tasks still require focused verification. Commit and push after each coherent slice.

---

## Current Baseline

- `v0.5` is tagged on `main` after Ask-in-English and local AI receipts.
- Packaging already exists through `script/build_and_run.sh --package`, producing `dist/LithePG.app` with a stripped executable.
- CI is manual-only (`workflow_dispatch`) because account billing/spending-limit settings previously blocked push/PR-triggered Actions.
- Apple signing/notarization credentials and Homebrew tap target are external decisions; scripts/docs can be prepared before those are available.

## Risks / Guardrails

- **Credential leakage:** signing/notary secrets stay in keychain/env only; no logs or docs include them.
- **Overclaiming launch readiness:** unsigned/unnotarized artifacts are local/dev only; docs must label them honestly.
- **CI blocked externally:** do not treat old manual-only CI failures as code failures without confirming current account settings.
- **Screenshot privacy:** only seeded/dummy data in screenshots.
- **Scope creep:** no updater, App Store, telemetry, or advanced AI in v1.0.
