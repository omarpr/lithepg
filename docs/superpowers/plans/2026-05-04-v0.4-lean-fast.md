# v0.4 — Lean & Fast — Implementation Plan

**Workflow:** Direct commits to `main`; no PRs unless Omar asks.

## Current Baseline

- v0.3 is dogfood-ready after the final results-pane vertical fill polish.
- Release binary is 21,909,208 bytes / 20.89 MiB on 2026-05-04, under the new 50 MiB hard cap and about 9.11 MiB away from the 30 MiB stretch goal.
- Local `psql` is available at `/opt/homebrew/opt/libpq/bin/psql`; the v0.4 harness uses it automatically when `psql` is not on `PATH`.

## Task 2: Binary-size reduction pass

**Goal:** Keep the release binary below the 50 MiB hard cap and push toward the 30 MiB stretch goal without weakening correctness/security.

Steps:
- [ ] Inspect release binary size contributors.
- [ ] Check build settings for accidental debug symbols or packaging overhead.
- [ ] Identify dependency/code paths that can be trimmed or deferred.
- [ ] Rebuild release binary after each meaningful reduction.
- [x] Turn CI size observation into a warning above 30 MiB and a fail gate above 50 MiB.
- [ ] Document trade-offs in `docs/dogfood-log.md`.

## Task 3: Cold-start reduction pass

**Goal:** Keep app launch under 500 ms to useful readiness.

Steps:
- [ ] Measure cold start before changes.
- [ ] Defer non-essential startup work where safe.
- [ ] Avoid blocking UI launch on persistence/schema/history loads unless needed.
- [ ] Record before/after measurements.

## Task 4: Query-path overhead pass

**Goal:** Keep simple localhost query overhead under 5 ms versus baseline after connection setup.

Steps:
- [ ] Measure persistent-connection `SELECT 1` median/p95.
- [ ] Compare against `psql` or the agreed temporary proxy.
- [ ] Optimize only measured overhead, not speculative paths.
- [ ] Keep all benchmark queries read-only.

## Task 5: Dogfood triage and stability window

**Goal:** Close v0.3 dogfood items and prove 7 crash-free days.

Steps:
- [ ] Triage each v0.3 dogfood issue as fixed/deferred/backlog.
- [ ] Track daily dogfood status in `docs/dogfood-log.md`.
- [ ] Do not tag `v0.4` until the 7-day zero-crash window is satisfied.
