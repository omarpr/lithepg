# v0.5 — AI-Ready — Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Spec:** `docs/superpowers/specs/2026-05-24-v0.5-ai-ready-design.md`

**Goal:** Add a private, local-first "Ask in English" path that drafts SQL from the current schema, with deterministic tests before any real model adapter work.

**Architecture:** Enrich schema metadata, build a local schema index, add an `AIQueryService` boundary with deterministic and local-model implementations, then wire a SwiftUI Ask palette that inserts generated SQL into the active tab for review. Keep generated SQL read-only-by-default and never auto-run.

**Tech Stack:** Swift 6.2, SwiftUI, Observation, Swift Testing, existing `LithePGCore`/`LithePGApp` targets. No new dependencies until the model-adapter task explicitly measures binary/runtime impact.

**Workflow:** Direct commits to `main`; no PRs unless Omar asks. Every code task follows TDD: write failing tests, verify RED, implement minimal GREEN, run target tests and then full `swift test` before commit.

---

## Risks / Guardrails

- **Model bloat:** Keep model artifacts external and measure runtime dependencies before merging.
- **Hallucinated SQL:** Insert drafts for review only; never auto-run; deterministic service should refuse unknown prompts.
- **Schema leakage:** No network calls, no cloud endpoints, no prompt/result logging by default.
- **Overbuilding:** Start lexical and deterministic; embeddings/model runtime comes after service seams and UX prove useful.
