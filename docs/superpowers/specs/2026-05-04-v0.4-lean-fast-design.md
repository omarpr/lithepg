# v0.4 — Lean & Fast — Design

**Roadmap source:** `docs/superpowers/specs/2026-04-18-roadmap-design.md` §6 v0.4.

## Goal

Make LithePG prove the thesis that a pure-Swift local Postgres client can feel lean, fast, and reliable enough to keep dogfooding after v0.3.

## Exit Criteria

- Release binary size is below a 50 MiB hard cap, measured in CI; 30 MiB remains the stretch goal.
- Cold start is below 500 ms on the maintainer's primary machine.
- Simple localhost query-path overhead is below 5 ms versus `psql` on the same Postgres container/baseline.
- Zero crashes across 7 days of daily dogfooding.
- Dogfood log items from v0.3 are triaged as fixed, deferred with rationale, or promoted to the v1.1+ backlog.

## Current Baseline — 2026-05-04

- Release binary: 21,909,208 bytes / 20.89 MiB via `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer swift build -c release --product LithePGApp`.
- Baseline is already under the new 50 MiB hard cap, with about 9.11 MiB left to reach the 30 MiB stretch goal.
- `psql` is available at `/opt/homebrew/opt/libpq/bin/psql`; the v0.4 harness uses it as the primary local query-overhead comparator.
- The dogfood log has one v0.3 UI item fixed immediately: the results pane/table now fills the allocated vertical space.

## Measurement Strategy

### Binary size

Use the existing CI release-build observation, then turn it into a CI warning above 30 MiB and a fail gate above 50 MiB. Track both bytes and MiB in `docs/dogfood-log.md`.

### Cold start

Add a small local measurement script that launches `LithePGApp` with the dogfood startup URL/query and records time to first rendered result or an explicit app-ready marker. Prefer app-level instrumentation over shell process timing so the metric reflects useful readiness, not just process spawn.

### Query overhead

Use the seeded local Postgres container from `script/dogfood_postgres.sh`. Benchmark a simple `SELECT 1` and the existing `lithepg_demo.customer_revenue` query through:

1. `psql` when available.
2. LithePG connector/AppState path.

Run enough iterations to ignore one-time connection setup and report median/p95. Keep the test read-only.

### Stability

Keep using `docs/dogfood-log.md` for daily dogfood incidents. Any crash gets a timestamp, query/context, stack/log pointer if available, and fix/defer decision.

## Non-Goals

- AI/NL2SQL work remains v0.5.
- Public launch packaging remains v1.0.
- Export formats beyond the current copy/export placeholders remain deferred unless dogfooding proves they are blocking.
