## Goal

Move LithePG from a capable query-experience prototype to a daily-driver candidate: saved connections, opt-in query history, environment awareness, and production safety cues.

## Scope

### In scope

- Saved connection metadata persisted locally:
  - display name
  - host, port, database, username
  - TLS mode / CA path reference
  - optional SSH target metadata
  - environment label (`development`, `staging`, `production`, `custom`)
  - secret reference identifier, not the secret itself
- Credential storage boundary:
  - passwords/secrets live behind a Keychain-facing abstraction
  - no password persisted in SwiftData/model metadata
  - test implementation uses an in-memory secret store
- Query history, opt-in:
  - disabled by default until user enables it
  - stores SQL text, connection/environment metadata, timestamp, elapsed time, row/command summary
  - clear-all operation
  - does not store result rows
- Production warning cues:
  - red/alert banner when connected to a production-tagged saved connection
  - clear connection label includes environment
- Dogfood protocol support:
  - continue maintaining `docs/dogfood-log.md`
  - record switches to other tools and missing features

### Out of scope

- Cloud sync or shared connection libraries
- Import/export of saved connections
- Full profile/account system
- Query result persistence
- Advanced environment policy rules beyond visible warnings

## Data Model

### SavedConnection

A lightweight metadata object, eventually SwiftData-backed, with no secret values.

Required fields:

- `id: UUID`
- `name: String`
- `host: String`
- `port: Int`
- `database: String`
- `username: String`
- `tlsMode: String`
- `pinnedRootCertificatePath: String?`
- `sshTarget: String?`
- `environment: ConnectionEnvironment`
- `secretReference: String?`
- timestamps

### QueryHistoryEntry

Stores useful recall data without result rows.

Required fields:

- `id: UUID`
- `connectionName: String?`
- `connectionLabel: String`
- `environment: ConnectionEnvironment?`
- `sql: String`
- `executedAt: Date`
- `elapsedMilliseconds: Int64`
- `summary: String`
- `succeeded: Bool`

## Service Boundaries

- `SavedConnectionStore`: CRUD for metadata.
- `CredentialStore`: save/load/delete secrets by stable reference.
- `QueryHistoryStore`: append/list/clear history.
- App UI talks to services through AppState or a small persistence coordinator.

## Exit Criteria Mapping

- Saved connections persisted: metadata store + Keychain abstraction wired for password save/load.
- Query history persisted: opt-in store + UI controls + clear-all.
- Multi-environment switching: saved connection list with visible environment labels.
- Production warning: red banner while connected to production-tagged connection.
- Dogfood bar starts: log maintained and v0.3 tag cut when usable.

## Risks

- SwiftData friction in SwiftPM executable/test target. Mitigation: keep model/service logic mostly framework-agnostic and cover with in-memory tests first.
- Keychain testability. Mitigation: protocol abstraction and in-memory fake in tests.
- Accidentally storing secrets. Mitigation: tests assert saved metadata contains no password field/value.
