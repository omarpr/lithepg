# v0.2a ‚Äî App Shell & Editor ‚Äî Design Spec

**Date:** 2026-04-18
**Status:** Approved design; implementation planning pending.
**Milestone:** First slice of v0.2 (Query Experience). Sibling slices `v0.2b` and `v0.2c` follow.

## 1. Context

v0.1 shipped a CLI (`lithepg`) that runs `SELECT 1` across plain / TLS / SSH-tunneled paths ‚Äî the walking skeleton that proved the pure-Swift, no-`libpq` thesis. Roadmap (`docs/superpowers/specs/2026-04-18-roadmap-design.md` ¬ß6) defines **v0.2 ‚Äî Query Experience** as the next milestone: make it nice to use.

v0.2 as originally scoped has six exit criteria spanning app shell, syntax-highlighted editor, results grid with pagination, schema tree, multi-tab workspace, keyboard-first navigation, and a binary-size CI gate. That's close to the upper edge of what fits in one spec. **v0.2 is split into three shippable sub-slices**; this spec covers the first, **v0.2a**. The umbrella `v0.2` git tag lands once all three sub-slices merge.

## 2. Scope

**v0.2a ships:** a SwiftUI macOS app that launches, prompts for connection info, opens a persistent connection, lets the user type SQL in a syntax-highlighted editor, runs it on Cmd-Enter, and shows rows in a basic table below. One window, one connection, one tab. The `v0.1` CLI binary remains in the package as a dev/smoke utility.

**Success criterion:** the maintainer launches the app, connects to their `minmaxing` Postgres, runs `SELECT version();`, and sees the server string rendered in a table ‚Äî no Terminal involved. Logged in `docs/dogfood-log.md` as the v0.2a receipt.

## 3. Non-Goals for v0.2a

Deferred to later v0.2 slices or beyond:

- Multi-tab workspace ‚Üí `v0.2c`.
- Schema tree sidebar ‚Üí `v0.2b`.
- Results pagination, column resize/sort, custom cell rendering ‚Üí `v0.2b`.
- Full keyboard-first navigation + documented shortcut surface ‚Üí `v0.2c`. v0.2a commits only **Cmd-Enter** (run) and **Cmd-.** (cancel).
- Binary-size CI gate ‚Üí `v0.2c`. v0.2a records the baseline size as an observation only.
- Saved connections, query history, Keychain integration ‚Üí `v0.3`.
- AI ‚Üí `v0.5`.
- Proper `.app` bundling + codesigning + notarization ‚Üí `v1.0`. v0.2a ships a pure SPM executable.

## 4. Architecture & Module Layout

Three SPM targets. No `.xcodeproj` (per `CLAUDE.md`).

- **`LithePGCore`** ‚Äî library. Holds `ConnectionConfig`, `PostgresConnector`, `SSHTunnel`, `QueryResult`, and the error-redaction helper (all moved out of the current `Sources/lithepg/`). No UI dependencies. Enforces the `ARCHITECTURE.md` boundary "Services never import SwiftUI."
- **`lithepg`** ‚Äî existing CLI executable. Depends on `LithePGCore`. Preserved verbatim as a dev/smoke utility; `swift run lithepg --url ...` keeps working. No new CLI surface in v0.2a.
- **`LithePGApp`** ‚Äî new SwiftUI executable. Depends on `LithePGCore` + Runestone. Runs via `swift run LithePGApp`.

**`LithePGApp` internal files (`Sources/LithePGApp/`):**

- `LithePGApp.swift` ‚Äî `@main` + SwiftUI `App` with a single `WindowGroup`.
- `AppState.swift` ‚Äî `@Observable` class; single source of truth for connection state, editor text, last result, last error, `isRunning`, and the active `Task` handle for cancellation.
- `ConnectSheet.swift` ‚Äî SwiftUI form: URL field + TLS toggle + TLS CA file picker + SSH toggle + SSH target field. Env-var autofill.
- `WorkspaceView.swift` ‚Äî vertical split: `EditorView` on top, `ResultsTable` on bottom, toolbar with Run/Cancel/Disconnect buttons.
- `EditorView.swift` ‚Äî `NSViewRepresentable` wrapping Runestone's `TextView`. Configures tree-sitter SQL grammar + a theme matching the system appearance.
- `ResultsTable.swift` ‚Äî SwiftUI `Table` over `[ResultRow]` for row-returning results; single status line for command tags.

**`LithePGCore` additions for persistent connections:**

- `PostgresConnector.open(config:) async throws` ‚Äî opens and holds the connection.
- `PostgresConnector.close() async` ‚Äî tears it down.
- `PostgresConnector.execute(_ sql: String) async throws -> QueryResult` ‚Äî runs a statement on the held connection.
- Existing `runSelect1(config:)` stays untouched for the CLI + v0.1 integration tests.

**Key invariants (preserved from `ARCHITECTURE.md`):**

- `LithePGApp` never imports `PostgresNIO` directly ‚Äî only via `LithePGCore`.
- `LithePGCore` never imports SwiftUI or Runestone.
- Exactly one persistent `PostgresConnector` per connected window; no per-query open/close.

## 5. Connect Flow & Connection Lifecycle

**First launch:** app boots to a centered `ConnectSheet` over an empty workspace. Nothing else is interactive until connected.

**ConnectSheet fields:**

- **URL** ‚Äî `postgres://user:pass@host:port/db`. Same parser as the CLI (`ConnectionConfig.init(url:)`).
- **TLS toggle** ‚Äî off by default. When on, reveals a **CA path** file picker (`.pem` / `.crt`).
- **SSH toggle** ‚Äî off by default. When on, reveals an **SSH target** field (`user@host[:port]`). Mutually exclusive with TLS; same gate the CLI enforces in v0.1.
- **Connect button** ‚Äî disabled until the URL parses cleanly. On click: inline spinner + "Connecting‚Ä¶"; success dismisses the sheet; failure shows an error banner inside the sheet with field values preserved.
- **Env-var autofill on sheet open:** if `POSTGRES_URL`, `POSTGRES_TLS_CA`, or `POSTGRES_SSH` are set, pre-fill the corresponding fields. Dogfooding ergonomic match to the CLI flag workflow without committing to saved connections.

**Connection lifecycle:**

- One persistent `PostgresConnector` per window, opened on Connect, closed on Disconnect or app quit.
- **Disconnect** button in the window toolbar tears down the connector and re-shows `ConnectSheet`.
- **Reconnect-on-error:** when a query fails with a connection-level error, `AppState` surfaces a **Reconnect** button next to the error banner. No auto-reconnect ‚Äî explicit user action only (silent reconnects mask real infrastructure issues, which defeats the dogfood log).
- **No persistence.** Connection info is in-memory only; quitting clears it. Saved connections + Keychain are v0.3.

**Window title** reflects connection state: `"LithePG"` when disconnected, `"user@host:port/db"` when connected.

## 6. Query Execution, Result Model & Errors

**Submitting a query:**

- **Cmd-Enter** runs the current editor buffer. Only shortcut v0.2a commits beyond Cancel.
- **Run button** in the toolbar mirrors Cmd-Enter for discoverability.
- **Cancel** is **Cmd-.** (Mac convention). Wires to `Task.cancel()`; `PostgresConnector.execute` honors cooperative cancellation.
- While running: the Run button shows a spinner and the Cancel affordance.

**`QueryResult` (defined in `LithePGCore`):**

```swift
public struct QueryResult: Sendable {
    public let columns: [Column]
    public let rows: [Row]
    public let rowCount: Int
    public let elapsed: Duration
    public let status: Status
    public let truncated: Bool

    public struct Column: Sendable {
        public let name: String
        public let typeName: String
    }

    public struct Row: Sendable, Identifiable {
        public let id: Int
        public let cells: [Cell]
    }

    public enum Cell: Sendable {
        case null
        case text(String)
    }

    public enum Status: Sendable {
        case rows
        case command(tag: String, affected: Int)
        case empty
    }
}
```

**Cell rendering (v0.2a):** every non-NULL value is stringified via PostgresNIO's built-in decoders. Bytea ‚Üí `"<bytea N bytes>"`. Arrays / JSON / timestamps ‚Üí canonical text form. No custom cell renderers.

**Non-returning statements** (`INSERT`, `UPDATE`, `CREATE`, ‚Ä¶) land in `Status.command(tag:, affected:)`. `ResultsTable` swaps to a single-line status such as `"INSERT 0 3 ‚Äî 3 rows affected (12 ms)"`.

**Row cap.** Hard cap at **10,000 rows** for v0.2a. Rows are streamed from the cursor and consumption stops once the cap is hit. The user's SQL is **never** rewritten with `LIMIT` ‚Äî that would change semantics (e.g., `COUNT(*)`). When `truncated == true`, `ResultsTable` shows a banner: *"Showing 10,000 of ‚â•10,000 rows. Pagination lands in v0.2b."*

**Error handling:**

- Query errors render in a red banner between editor and results, showing PG `SQLSTATE`, message, and position when present.
- The existing `redactCredentials` helper from v0.1 runs on every error string before display.
- Connection-lost errors surface a **Reconnect** button as described in ¬ß5.
- The error banner persists until the user runs another query or edits the editor buffer.

## 7. Dependencies

**New top-level dependency in `Package.swift`:**

- **Runestone** ‚Äî NSTextView-based editor with tree-sitter integration. Transitive: tree-sitter C core + whatever language bundles we enable.

**SQL grammar:** via Runestone's tree-sitter language bundles. The implementation plan picks the specific grammar (generic `tree-sitter-sql` likely sufficient for v0.2a; PG-dialect-specific grammars are revisited if highlighting feels off during dogfooding).

Runestone introduces vendored C (tree-sitter core + grammar) into the dependency graph. This tensions with `CLAUDE.md`'s "no C shims" rule but matches the existing `swift-nio-ssl` / BoringSSL precedent documented in `TECH_STACK.md` ¬ß3. The plan updates `TECH_STACK.md` ¬ß3 to record the second vendored-C trade-off explicitly, by the same rule: authored-by-LithePG C shims remain forbidden; trusted-upstream vendored C is a conscious trade-off.

## 8. Binary-Size Handling

Observation only in v0.2a; the gate comes in v0.2c.

- CI records `ls -l .build/release/LithePGApp` on every run and prints it to the log.
- The v0.2a release binary size is recorded in `docs/dogfood-log.md` as the **baseline** for the v0.2c gate.
- The build does **not** fail on size in v0.2a.

## 9. Testing Strategy

- **`LithePGCore` unit tests (new):** `QueryResult` formatting ‚Äî NULL rendering, cell stringification, truncation-flag semantics, `Status.command` decoding, `execute(_:)` error mapping. No Postgres required.
- **`LithePGCore` integration tests (extend existing):** `POSTGRES_TEST_URL`-gated test runs `SELECT`, `INSERT`, and an intentionally-failing statement through the new `execute(_:)` path.
- **`LithePGApp` view-model tests (new):** `AppState` transitions ‚Äî connect ‚Üí workspace, run ‚Üí result, run ‚Üí error, cancel ‚Üí idle, disconnect ‚Üí sheet. No SwiftUI imports needed; `@Observable` is testable as a plain class.
- **Manual UI smoke:** documented in `docs/dogfood-log.md` as the v0.2a exit-criteria receipt.
- **CI unchanged in shape:** the existing `.github/workflows/ci.yml` already runs `swift build && swift test`. It automatically picks up `LithePGApp` and `LithePGCore`. The binary-size observation line is the only workflow addition.

## 10. Exit Criteria

1. `swift run LithePGApp` launches the app; `ConnectSheet` appears.
2. Connecting with each of the three v0.1-proven paths succeeds (plain, TLS, SSH-tunneled) ‚Äî visible in window title.
3. Editor displays syntax-highlighted SQL.
4. Cmd-Enter / Run executes the query; result rows render in a table, or a command-tag status renders for non-returning statements.
5. Cmd-. cancels a running query.
6. Query errors render inline with PG code + message (credentials redacted via existing helper).
7. Disconnect returns to `ConnectSheet`; app quit closes the connection cleanly.
8. Maintainer runs `SELECT version();` against `minmaxing` Postgres entirely in the app ‚Äî no Terminal. Logged in `docs/dogfood-log.md` as v0.2a receipt.
9. `swift test` green. All v0.1 tests still pass. New `LithePGCore` + `AppState` tests pass.
10. Release-build binary size recorded in dogfood log as v0.2c-gate baseline.

## 11. Risks

| # | Risk | When we find out | Mitigation |
|---|------|------------------|------------|
| 1 | Runestone's macOS support is iOS-first; `NSViewRepresentable` integration has rough edges (cursor color, first-responder focus, system dark-mode). | v0.2a implementation | Budget a pass to polish focus/cursor behavior. Fall back to a thinner `NSTextView` wrapper with a hand-rolled regex highlighter if Runestone fights us ‚Äî we lose incremental parsing quality, not app functionality. |
| 2 | tree-sitter-sql grammar produces wrong highlighting on PostgreSQL-specific syntax (dollar-quoted strings, `plpgsql`, array literals). | v0.2a dogfooding | Dogfood-log the miss. Switch grammar to a PG-dialect fork if one exists; otherwise file as v0.2b polish item. |
| 3 | Binary size spikes once Runestone + tree-sitter land, eating headroom for v0.2b (schema tree) + v0.2c (tabs) against the v0.4 <15MB target. | v0.2a exit | Record the baseline explicitly in the dogfood log. If the baseline is alarming, promote the v0.2c size gate to v0.2b or adjust the v0.4 target with documented rationale. |
| 4 | Persistent connections introduce lifecycle surprises (idle timeouts, server-side session termination, FD leaks on app crash). | v0.2a dogfooding | Surface connection errors with the Reconnect affordance (already specified). Add `PostgresConnector.close()` to app termination lifecycle. Log any FD-leak-shaped regression in the dogfood log. |
