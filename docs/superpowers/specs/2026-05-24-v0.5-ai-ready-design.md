# v0.5 — AI-Ready — Design

**Roadmap source:** `docs/superpowers/specs/2026-04-18-roadmap-design.md` §6 v0.5.

## Goal

Make LithePG's local-first AI path real without compromising the v0.4 thesis: keep the app lean, private, and fast while adding an "Ask in English" workflow that can draft runnable SQL from the current database schema.

## Exit Criteria

- Local schema introspection produces an on-device schema index suitable for retrieval and prompt construction.
- On-device NL2SQL path exists behind a service boundary that can use CoreML or MLX models, with deterministic offline tests that do not require a downloaded model.
- Command palette / "Ask in English" UI produces runnable SQL for:
  - Simple single-table queries.
  - 2-table joins using foreign-key relationships.
- Generated SQL is inserted into the active editor for review and is never auto-run.
- AI model artifacts ship as a separate download/imported bundle, not inside the app binary.
- No prompts, schemas, query results, credentials, or history leave the device.
- v0.4 gates stay green: app binary remains below the 50 MiB hard cap, cold start remains below 500 ms, and the default/local dogfood checks pass.

## Current Baseline — 2026-05-24

- `v0.4` is tagged on `main` after a green local dogfood check and elapsed seven-day zero-crash stability window.
- Current app surface includes a persistent connection, schema sidebar, tabbed editor, query history, saved connections, and production warning banner.
- `DatabaseSchema` currently models schemas, relations, and columns. It does not yet model primary keys, foreign keys, comments, row-count estimates, or semantic aliases.
- `SchemaIntrospector` is the right boundary to enrich schema metadata. `AppState.refreshSchema()` already maintains the live schema snapshot the AI layer should consume.
- `WorkspaceView` toolbar is the natural place to expose a command palette / "Ask" affordance.
- No CoreML/MLX dependencies are in `Package.swift`; adding inference dependencies must be measured against the binary budget and should be deferred until the service boundary and deterministic tests are in place.

## Product Behavior

### Ask flow

1. User opens "Ask" from the toolbar or keyboard shortcut.
2. User types a natural-language request, e.g. "show the top customers by revenue".
3. LithePG builds a local context from the current schema index and active connection metadata.
4. AI service returns a draft SQL statement plus a short explanation and confidence/status.
5. UI inserts the SQL into the active editor, replacing selected text or appending to the active tab.
6. User reviews and runs with the existing `⌘↩` command.

### Safety defaults

- Generated SQL is never auto-executed.
- Destructive statements should be refused in the v0.5 UI because this milestone is read-only query drafting.
- Production connections keep the existing production warning visible while Ask is open.
- If no schema is loaded, Ask offers to refresh schema first rather than hallucinating from nothing.
- If no local model is available, the UI shows setup/import instructions and can still run deterministic test fixtures in development.

## Architecture

```text
WorkspaceView
  └─ AskQueryView / command palette
       └─ AppState.askInEnglish(_:) async
            └─ AIQueryService protocol
                 ├─ DeterministicAIQueryService (tests/dev fixtures)
                 └─ LocalModelAIQueryService (CoreML/MLX adapter)
                      └─ SchemaIndex + prompt builder
```

### Schema index

Create a lightweight on-device index from `DatabaseSchema`:

- Relation document: schema name, relation name, kind, columns, likely display labels.
- Column document: qualified column path, type, nullability, default, relation context.
- Relationship document: foreign-key edge from child relation/columns to parent relation/columns.

v0.5 can start with lexical scoring over schema documents before adding embeddings. The service boundary should allow a vector-backed implementation later without blocking UI/testing.

### NL2SQL service boundary

`AIQueryService` should be async and local-only:

```swift
public protocol AIQueryService: Sendable {
  func draftSQL(for request: AIQueryRequest) async throws -> AIQueryDraft
}
```

Where `AIQueryRequest` includes the user prompt, schema index snapshot, selected connection environment, and optional query-history hints that are enabled only if the user opted into history. `AIQueryDraft` includes SQL, explanation, referenced schema objects, and a confidence/status enum.

### Model artifact strategy

- Do not bundle a model in the app executable.
- Add a local model registry that can point at a user-provided model bundle/path under Application Support.
- Keep model download/import UI minimal in v0.5; public distribution polish belongs closer to v1.0.
- Any model runtime dependency must be measured for binary and startup impact before merging.

## Privacy and Security Invariants

- No network calls from the AI layer.
- No cloud inference endpoint, telemetry, or schema upload.
- Credentials and passwords never enter prompts, logs, schema documents, or generated explanations.
- Query results are not included in AI context for v0.5 unless explicitly designed later.
- Prompt/debug logging is disabled by default and, if added, must redact connection labels and schema-sensitive details or be dev-only.

## Testing Strategy

- Unit tests for schema enrichment and relationship extraction.
- Unit tests for schema-index document generation and lexical retrieval.
- Unit tests for deterministic NL2SQL outputs from fixture schemas.
- AppState tests for Ask flow: no schema, successful draft insertion, model unavailable, and production connection safety behavior.
- View-model/headless tests first; avoid UI automation unless the toolbar/palette cannot be verified otherwise.
- Existing dogfood check must remain green.

## Non-Goals

- Multi-table joins beyond two tables.
- Query optimization suggestions.
- Automatic execution of generated SQL.
- Cloud/provider-backed inference.
- Shipping a large bundled model in the app binary.
- Full model downloader UX, signed model catalogs, or update channels.

## Open Decisions

- CoreML vs MLX first adapter: decided for v0.5 Task 9 in favor of a minimal CoreML scaffold. CoreML comes from the macOS SDK, adds no Swift package dependency, and measured as a +21,248 byte / +0.020 MiB release-binary delta with no bundled model. MLX remains a future option if a model candidate justifies the package/runtime cost.
- Embeddings first vs lexical retrieval first: prefer lexical first for v0.5 to keep implementation small and testable, then swap behind the same index protocol if needed.
- Model candidate and quantization format: still open. v0.5 loads only an env-gated external CoreML artifact smoke path; NL2SQL model I/O mapping is intentionally deferred until a reviewed model contract exists.
