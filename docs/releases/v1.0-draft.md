## Release title

LithePG v1.0 — Public Launch

## GitHub release body draft

LithePG v1.0 is the public-launch release for a lean, Mac-native PostgreSQL client with local-first SQL drafting.

### Highlights

- Native macOS PostgreSQL client built in Swift with `postgres-nio` and no `libpq` dependency.
- Saved connection metadata with passwords stored separately in the macOS Keychain.
- Schema sidebar, tabbed SQL workspace, results grid, pagination, query history, and keyboard shortcuts.
- Ask-in-English SQL drafting that uses local schema context, inserts SQL for human review, and never auto-runs generated SQL.
- Light, dark, and system appearance settings, with dark as the default.
- Public project docs: changelog, security policy, governance, contribution guide, code of conduct, and issue/PR templates.
- Signed and notarized macOS app distribution path with Homebrew cask metadata prepared.

### Install

Download `LithePG.app.zip` from the assets below, unzip it, move `LithePG.app` to `/Applications`, and open it from Finder.

### Local-first AI and privacy posture

LithePG does not send prompts, schema metadata, credentials, query history, or query results to cloud AI services. The default Ask flow is deterministic and local. A CoreML adapter scaffold exists for user-provided local model artifacts, but model artifacts are not bundled and are disabled unless explicitly configured.

Generated SQL is a draft: it is inserted into the editor for review and is not executed automatically.

### Known launch notes

- The app is macOS-only.
- AI model artifacts are separate user-provided downloads/experiments, not bundled assets.
- Advanced AI joins beyond the v0.5 two-table receipt, ERD visualization, DBA helpers, extra export formats, and auto-updates are deferred to v1.1+.
